# FastAPI Developer Portfolio Backend

Асинхронное серверное приложение на базе фреймворка FastAPI, предназначенное для обработки форм обратной связи веб-сайта-портфолио. Проект интегрирован с языковыми моделями (LLM) через API-провайдеров для автоматического анализа тональности входящих сообщений и генерации мгновенных ответов, а также оснащен системой двухстороннего Email-оповещения.

## Архитектура и структура проекта

Приложение спроектировано в соответствии со слоистой архитектурой (Layered Architecture) для обеспечения изоляции ответственности компонентов:

1. **API Слой (app/api/endpoints/):** Отвечает за маршрутизацию HTTP-запросов, валидацию входящих DTO посредством Pydantic и формирование унифицированных ответов.
2. **Сервисный слой (app/services/):** Реализует бизнес-логику приложения, координирует взаимодействие между ИИ-модулем, почтовым сервисом и репозиторием.
3. **Слой данных (app/repositories/):** Взаимодействует с репозиторием базы данных SQLite, используя SQLAlchemy ORM для выполнения CRUD-операций.
4. **Инфраструктурный слой (app/core/, app/middlewares/):** Содержит конфигурационные файлы, кастомные логгеры, HTML-шаблоны писем и компоненты защиты (Rate Limiting).

## Функциональные возможности

* **Валидация данных:** Строгая проверка типов входных параметров (Email, Телефон, Имя) на уровне Pydantic. Возврат статус-кода 422 при несоответствии схем.
* **Логирование запросов:** Middleware-компонент регистрирует метаданные каждого запроса (метод, путь, HTTP-статус, время выполнения в миллисекундах) в локальный файл `data/app.log`.
* **Глобальная обработка исключений:** Перехват любых необработанных ошибок на уровне приложения с записью Stack Trace в лог и возвратом клиенту стандартизированного ответа 500 Internal Server Error.
* **Анализ тональности и авто-ответ (AI):** Интеграция с моделями через OpenAI SDK в режиме JSON Mode. Система извлекает тип тональности (positive, neutral, negative) и генерирует контекстный ответ.
* **Отказоустойчивость (Graceful Fallback):** При сбое внешнего API искусственного интеллекта (лимиты, невалидный токен, сетевой таймаут) ошибка логируется, а система подставляет дефолтные нейтральные значения, сохраняя транзакцию в БД и не прерывая пользовательский сценарий.
* **Асинхронные Email-уведомления:** Отправка кастомизированных HTML-писем администратору (уведомление о заявке) и пользователю (копия обращения со встроенным ответом ИИ) с выносом блокирующих операций ввода-вывода в пул потоков через `run_in_threadpool`.
* **Ограничение частоты запросов (Rate Limiting):** Защита эндпоинта отправки формы от спама. Ограничение составляет 3 запроса в минуту с одного IP-адреса (возврат 429 Too Many Requests).
* **Мониторинг:** Эндпоинты `/api/health` для проверки статуса работоспособности сервера и `/api/metrics` для агрегации статистических данных по обращениям напрямую из SQLite.

## Отчет об использовании AI при разработке

Проектирование и написание кодовой базы выполнялось в синергии с искусственным интеллектом, выступавшим в роли архитектурного ментора.

### Сферы применения ИИ:
1. **Трансляция архитектурных паттернов:** Перенос концепций Dependency Injection и слоистой структуры из экосистемы Node.js (NestJS) на рельсы Python (FastAPI).
2. **Промпт-инжиниринг:** Разработка системных инструкций для LLM с целью обеспечения гарантированного вывода валидного JSON-объекта без Markdown-разметки, что исключает сбои при парсинге через десериализатор.
3. **Оптимизация SQL-запросов:** Формирование агрегирующих запросов SQLAlchemy с группировкой (GROUP BY) для построения динамического эндпоинта метрик.

### Методология верификации результатов:
* Ручное тестирование граничных сценариев через интерактивную документацию Swagger UI.
* Статический анализ типов в среде разработки VS Code посредством Pylance.
* Имитация аппаратных и программных сбоев (подмена токенов API, отключение сети) для верификации блоков обработки исключений в контуре интеграции с ИИ.
